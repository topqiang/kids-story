# Codex 中国开发者默认规则

## 基本原则

- 所有代码注释和文档使用中文编写。
- 遵循中国软件开发行业标准和最佳实践。
- 优先考虑国内常用的技术栈和工具链。
- 考虑国内网络环境和资源访问限制。
- 每一次协作都必须更新 `CHANGELOG.md`。

## 代码规范

- 使用 UTF-8 编码。
- JavaScript、TypeScript、JSON 统一使用 2 空格缩进。
- Python、Java 统一使用 4 空格缩进。
- 文件名使用小写字母和下划线命名法。
- 变量和函数名使用驼峰命名法。
- 常量使用大写字母和下划线命名法。

## 国际化和本地化

- 默认使用中国时区 CST，UTC+8。
- 数字、日期和文案表达遵循中国用户习惯。
- 默认中文本地化，并为后续国际化预留接口。

## 安全要求

- 遵循国内网络安全法规。
- 实施必要的数据加密和隐私保护措施。
- 做好输入验证和注入防护。
- 重要设计应考虑等保要求。

## 性能优化

- 考虑国内网络特点进行 CDN 和静态资源配置。
- 优化首屏加载时间。
- 压缩图片、视频和静态资源。
- 注意数据库查询和素材读取效率。

## 文档要求

- 提供详细中文技术文档。
- 包含部署、制作、运维和复盘说明。
- API、工具、模板和流程说明均使用中文描述。
- 注释清晰，便于团队协作。

## 本项目 Codex 启动顺序

1. 先阅读 `README.md`，了解项目定位和目录结构。
2. 再阅读 `codex/memory.md`，继承项目记忆、业务进展和当前状态。
3. 如需继续创作、审稿或制作短片，阅读 `codex/skills/kids_story.md`。
4. 继续任何协作前，阅读 `docs/series_bible.md` 和 `production/quality_checklist.md`。
5. 产出新内容后，同步更新相关剧集、制作资料和 `CHANGELOG.md`。
