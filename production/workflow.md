# 制作流程

## 阶段一：策划

1. 从 `docs/topic_bank.md` 选择选题。
2. 使用 `templates/episode_template.md` 创建单集文件。
3. 写清楚目标共鸣点、主要笑点和风险检查。
4. 控制剧情为一个核心冲突，不扩展副线。
5. 在 `.log/` 下创建本次任务生产日志，记录任务目标、会话时间和后续产物路径。

## 阶段二：剧本

1. 用 3 秒开场句制造冲突。
2. 对白控制为短句，适合配音和字幕。
3. 每 15-20 秒设置一个动作或表情变化。
4. 结尾用反转或温情收束，不硬讲道理。
5. 将剧本生成或返修使用的完整输入提示词、关键约束和确认结果写入 `.log/`。

## 阶段三：分镜

1. 单集建议 6-10 个镜头。
2. 每个镜头必须包含画面、动作、对白、音效和时长。
3. 优先使用中近景、特写和轻微推拉镜头。
4. 同一场景内保持角色站位稳定，减少制作复杂度。
5. 将分镜生成或返修使用的完整提示词、角色参考和连续性约束写入 `.log/`。

## 阶段四：画面

1. 先确定角色标准图和表情包。
2. 每集生成关键帧，再补充必要过渡帧。
3. 背景简洁，不抢角色表演。
4. 人工检查角色发型、服装、脸型和颜色一致性。
5. 记录每次图片生成的完整提示词、参考图路径、模型、尺寸、种子、关键参数和结果路径。
6. 如调用三方图片接口，记录脱敏后的请求入参、接口状态、响应摘要、错误信息和输出文件路径。

## 阶段五：声音

1. 先录制或生成对白，再匹配口型和动作。
2. 每集加入 3-5 个音效点。
3. 背景音乐音量低于对白，避免盖字。
4. 导出前统一响度，避免忽大忽小。
5. 记录配音、音效和音乐生成提示词、声音参数、接口状态、响应摘要和输出音频路径。

## 阶段六：剪辑

1. 画幅使用 1080x1920。
2. 字幕放在底部安全区，每行不超过 14 个中文字符。
3. 开头不加长片头，直接进入冲突。
4. 结尾保留 0.5-1 秒反应时间。
5. 记录视频生成、剪辑、转场、字幕和导出参数；如调用三方视频接口，记录脱敏后的请求与响应信息。

## 阶段七：审核

1. 按 `production/quality_checklist.md` 完成安全、视觉、声音和平台检查。
2. 发现危险模仿、恐吓教育或表达不清时必须返修。
3. 发布后使用 `templates/review_template.md` 记录数据。
4. 若产物不符合预期，将问题表现、可能原因和下一轮建议调整的提示词或参数补充到 `.log/`。
